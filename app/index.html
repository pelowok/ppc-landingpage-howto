Deploying LogicNow PPC Responsive-design Landing Pages via HubSpot

  1. Stakeholders and requirements
    1. Identify stakeholders
    2. Gather requirements
      1. Stakeholder goals
      2. Delivery schedule
      3. Technology requirements
    3. Determine target personas
    4. Write it all up
    5. Signoff
      1. Requirements are now locked
  2. Research
    1. Research best practices
    2. Research competition
    3. Gather branding guides
    4. Gather required assets
      1. Pre-existing [required] artwork
      2. Pre-existing [required] typefaces
      3. Software and technology to be used
  3. Design
    1. Persona area map
      1. Identify persona characteristics for map
        1. Demographic information
        2. Job and seniority
        3. Pain points
        4. Values and goals
        5. Experience they are hoping for
        6. Most common objections
      2. Apply weights/metrics to persona characteristics
      3. Determine content ratios based on weights
      4. Create weighted content wireframe (persona area map)
    2. Wireframe
      1. Translate persona area map to wireframe
        1. Break persona area map down to distributed percentage blocks
        2. Layout blocks for responsive layouts
          1. Can you get data on screen resolutions hitting landing pages?
          2. Responsive design is driven by a mobile-first approach, meaning layouts will be designed for the smallest screen size, and then will only change when the design stretches enough that it 'breaks'
          3. Can this project follow a mobile-first approach, or does it need to defer to a larger format as its base size?
          4. Set a min-width for the UIWebView
          5. !important! Layout order is from smallest required webview to largest
      2. Translate area wireframes into content-block wireframes
        1. Split percentage-blocks of content type into distributed units
        2. At any given time, above or below the fold, you should see roughly the same percentage of content types
        3. The more text-heavy the content, the lower it should fall on the page
        4. Meet common objections by stating them as education/talking points - don't go negative
      3. Apply mock content
        1. Color denotes content types only, otherwise, black and white
        2. Placeholders, not images
        3. Lorem Ipsum, no actual content
      4. Stakeholders review and sign-off on layout
        1. Check wireframe against requirements from Section I
        2. Stakeholders review wireframe individually, not as group, to confirm their specific requirements are met
        3. Layout is now locked
        4. Content types are now locked
        5. Wireframe is now locked for all required webview sizes
    3. Create landing page template
      1. Initialize development environment
        1. Initialize flexible grid framework
          1. Bootstrap 2.x is default for HubSpot COS
          2. Bootstrap 3.x can be implemented with a an attached .min. script
          3. Because Bootstrap loads regardless, the grid will have to be a presentation/front-end framework
          4. Be aware Bootstrap 2.x is going to load regardless and will need to be overridden at runtime
          5. Foundation 3.x
          6. Bootstrap
          7. Skeleton
        2. Style tools : CSS? LESS? SASS?
        3. Build tools : Bower? Grunt? Gulp? Node? Require?
        4. Code packager : minify.js? uglify.js? optimize.ly?
        5. Unit tests : qUnit? Karma? Swarm? Jasmine?
      2. Choose jQuery library based on development packages
        1. HubSpot runs jQuery 1.7.1 as part of its COS
        2. If you load in a later version of jQuery, you must do it after {% standard_header_includes %}
        3. You can't run without jQuery, so you must pick a version, so you can test through it
      3. Build flexible grid to match wireframes
        1. Use mock content
          1. It may save time to build using data bindings, through a framework like Angular
          2. Bindings later can be swapped out or simply draw data from an external source
          3. Don't bother with the form at this stage, they are currently built from global modules
        2. Apply styling in an over-ride style sheet
        3. Use branded typefaces
        4. Use branded colors in style library
        5. Build smallest wireframe [smallest webview] first
